Naming & data type rules

Tag name conventions

  • TitleCase with no spaces, underscores, or special characters
  • Use Left and Right instead of East, West, North, or South
  • Put the delineation at the end: FormerHeight1 not Former1Height
  • Be descriptive but succinct: BlenderSpeedHzCL2 not CL2BlenderSpeed

BIT data type is not permitted

BITs cause conflicting display patterns in downstream applications. Use INT instead — it stores and displays the same on/off information.
