Naming & Data Type Rules

Tag Name Rules

  • Use underscores instead of spaces
  • Start with a letter, not a number or special character
  • Avoid using reserved words like "select" or "table"
  • Be descriptive but concise

Data Type Rules

  • Use standard PLC data types (e.g. REAL, INT, BOOL)
  • Avoid using vendor-specific or custom data types
  • Ensure the data type matches the expected value range and format
  • Refer to the provided data type mapping for guidance
